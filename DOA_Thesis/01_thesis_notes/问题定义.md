# 问题定义

本文档是论文和代码实验的最高约束。后续写作、代码修改、训练和画图都必须先满足这里的定义；旧代码、旧权重、旧结果只能作为参考，不能直接作为论文证据。

## 1. 固定题目

低信噪比条件下基于坐标注意力网络的 DOA 估计方法研究

## 2. 一句话主线

针对低信噪比条件下传统 DOA 估计方法受噪声影响、样本协方差矩阵子空间结构退化的问题，本文构造多通道协方差矩阵输入特征，引入坐标注意力机制增强阵元相关结构表达，并通过空间谱输出和峰值搜索实现多信源 DOA 估计。

## 3. 问题锚点

### 3.1 研究对象

均匀线阵下的低信噪比多信源 DOA 估计。

### 3.2 必须解决的瓶颈

在低 SNR、有限快拍条件下，样本协方差矩阵受到噪声扰动，传统 MUSIC/ESPRIT 依赖的信号子空间与噪声子空间分离变差，导致空间谱峰偏移、漏检或角度估计误差增大。

本文要解决的不是“设计一个更复杂的神经网络”，而是：

> 如何利用协方差矩阵中仍然存在的阵元相关结构，在低 SNR 条件下得到更稳定的空间谱峰值和更低的 DOA 估计误差。

### 3.3 非目标

以下内容不作为本文主线：

- 宽带 DOA；
- 相干源解相干；
- 二维 DOA；
- 实测数据系统；
- 阵列失配鲁棒性主问题；
- Transformer/迁移学习/大模型方法；
- 追求任意场景下最优性能。

这些内容最多放在第 5 章展望。

| 项目 | 本文固定设置 | 说明 |
| --- | --- | --- |
| 阵列类型 | ULA | 均匀线阵 |
| 信源模型 | 窄带、远场、非相干 | 不研究相干源 |

## 5. 方法定义

### 5.1 输入

以接收信号的样本协方差矩阵为基础构造输入特征。

主方法建议采用四通道：

```text
X = [Re(R_hat), Im(R_hat), sin(angle(R_hat)), cos(angle(R_hat))]
```

其中 `R_hat` 为样本协方差矩阵。

2 通道输入：

```text
X_2ch = [Re(R_hat), Im(R_hat)]
```

只能作为消融基线，不作为完整方法。

### 5.2 网络

主方法暂定为 CA-DOA-Net：

```text
四通道协方差特征
    -> CNN 特征提取
    -> 坐标注意力模块
    -> 空间谱输出
```

坐标注意力的论文表述应聚焦在：

> 沿协方差矩阵行、列方向编码阵元相关结构，使网络在低 SNR 下更关注稳定的结构性特征，而不是把它写成通用注意力堆叠。

### 5.3 输出与后处理

网络输出角度网格上的空间谱，最终 DOA 由峰值搜索得到。论文中需要明确：

- 角度网格范围；
- 网格间隔；
- 峰值搜索策略；
- 最小峰间隔或 NMS 设置；
- 多信源匹配规则。

## 6. 创新点表述

创新点按重要性排序：

1. 面向低 SNR DOA 估计的多通道协方差矩阵特征构造。
2. 将坐标注意力引入协方差矩阵特征建模，增强行、列方向阵元相关结构表达。
3. 基于空间谱输出与峰值搜索实现多信源 DOA 估计，并通过谱形约束改善峰值可分性。

谨慎表述：

- 不写“首次提出”。
- 不把“物理约束损失”作为题目核心。
- 如果分离约束依赖峰值搜索或不可导后处理，只写成辅助正则或实验项，不写成严格端到端物理约束。

## 7. 验收标准

### 7.1 论文方向验收

只有同时满足以下条件，方向才算固定：

- 第 1 章能清楚说明为什么低 SNR 是问题；
- 第 2 章能说明 MUSIC/ESPRIT 依赖协方差矩阵和子空间分解；
- 第 3 章能把四通道输入、坐标注意力、空间谱输出讲成一个闭环；
- 第 4 章至少有主实验和两个消融实验支撑。

### 7.2 方法验收

方法必须回答四个问题：

| 问题 | 必须给出的答案 |
| --- | --- |
| 为什么不用原始信号输入？ | 本文聚焦协方差矩阵中的阵元相关结构 |
| 为什么要四通道？ | 实部、虚部与相位信息互补 |
| 为什么是坐标注意力？ | 行、列方向对应协方差矩阵中的阵元相关结构 |
| 为什么输出空间谱？ | 多信源估计可通过谱峰搜索统一得到 |

### 7.3 实验验收

最低必须完成：

| 实验 | 验收内容 |
| --- | --- |
| SNR 主实验 | CA-DOA-Net 在低 SNR 下相对 MUSIC/ESPRIT/CNN 有可解释优势 |
| 注意力消融 | CNN vs CNN + CA，证明提升来自坐标注意力 |
| 输入消融 | 2 通道 vs 4 通道，证明四通道输入有效 |
| 结果稳定性 | 至少固定随机种子；最好给 3 个 seed 的均值/方差 |

如果 CA-DOA-Net 效果不好，也不能硬写优势。需要改成：

> 本文验证了坐标注意力在该仿真设置下对协方差特征建模的影响，并分析其有效条件和失效原因。

## 8. 旧代码与旧结果处理原则

你已经说明旧代码“不一定对、效果不好”。因此从现在开始：

1. 旧权重不直接作为论文结果。
2. 旧测试图不直接进入第 4 章。
3. 旧代码只能用于定位已有实现思路。
4. 每个能写进论文的结果必须重新满足：
   - 配置可追踪；
   - 数据生成逻辑明确；
   - 训练命令明确；
   - 测试命令明确；
   - 指标计算规则明确；
   - CSV 与图可复现。

## 9. 下一步代码准入检查

在继续训练前，先检查并修正以下内容：

1. 数据生成是否符合第 4 节研究边界。
2. 2 通道、4 通道输入是否只差输入特征，不混入其他变量。
3. CNN、CNN+CA、完整方法的网络容量是否可比。
4. loss 设置是否能清楚对应 Exp1-Exp5。
5. peak search 和 RMSE/SR 计算是否统一。
6. train/test 是否保存完整 config、seed、checkpoint、CSV、图。

只有这些通过，才开始重新跑最小实验闭环。
