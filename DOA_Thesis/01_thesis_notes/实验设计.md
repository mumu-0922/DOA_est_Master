# 实验设计

## 0. 前置原则

旧代码、旧权重、旧测试图只作为参考，不直接进入论文。所有第 4 章结果必须重新生成，并且满足：

- 配置可追踪；
- 随机种子可追踪；
- 数据生成逻辑可解释；
- 训练命令可复现；
- 测试命令可复现；
- 指标计算规则统一；
- CSV 和图片同时保存。

在重新训练前，必须先做一次代码准入检查。

## 1. 总体验收目标

第 4 章至少证明三件事：

1. 低 SNR 下本文方法是否有效。
2. 坐标注意力模块是否有效。
3. 四通道输入是否有效。

注意：如果实验结果不好，也不能硬写“有效”。应改为分析有效条件和失效原因。

## 2. 固定参数

| 参数 | 设置 | 备注 |
| --- | --- | --- |
| 阵列 | ULA | 均匀线阵 |
| M | 8 | 主实验固定 |
| K | 3 | 主实验固定 |
| 快拍数 | 200 | 少快拍另做补充 |
| 角度范围 | [-60 deg, 60 deg] | 与网格输出一致 |
| 网格间隔 | 1 deg | 若使用 0.5 deg 必须单独成组 |
| 最小角间隔 | 5 deg 或 8 deg | 同一组实验不能混用 |
| 低 SNR 训练范围 | [-20, 0] dB 或 [-20, 5] dB | 必须写入 config |
| 测试 SNR | -20, -15, -10, -5, 0, 5, 10 dB | 主曲线 |
| 评价指标 | RMSE, Success Rate | 定义必须统一 |

## 3. 代码准入检查

训练前先检查这些问题：

| 检查项 | 通过标准 |
| --- | --- |
| 数据生成 | ULA、M=8、K=3、T=200、角度范围和 SNR 范围与论文一致 |
| 输入特征 | 2 通道和 4 通道只差输入特征，不混入其他设置变化 |
| 模型对比 | CNN、CNN+CA、完整方法的容量差异可解释 |
| 损失函数 | Exp1-Exp5 的 loss 设置能一一对应 |
| 峰值搜索 | 所有方法使用统一角度匹配和容差规则 |
| 指标 | RMSE 与 Success Rate 的定义固定 |
| 保存 | config、seed、best_model、CSV、图全部保存 |

代码准入不通过时，先改代码，不跑正式实验。

## 5. 最小实验闭环

先不要直接跑大训练。最小闭环只验证流程：

| 步骤 | 目标 |
| --- | --- |
| 1 | 用小训练集跑通 2 通道 CNN |
| 2 | 用小训练集跑通 4 通道 CNN |
| 3 | 用小训练集跑通 4 通道 CNN + CA |
| 4 | 确认每个实验都保存 config、checkpoint、CSV |
| 5 | 确认同一个测试脚本能输出 RMSE/SR |
| 6 | 确认画图脚本能生成主图 |

最小闭环通过后，才跑正式实验。

## 7. 结果归档规则

每次正式实验至少保存：

- `config.txt` 或 `config.json`
- `best_model.pth`
- 原始指标 CSV
- RMSE 曲线图
- Success Rate 曲线图
- 记录该结果用于论文哪个图表
